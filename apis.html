API's

On this page

Resumo das APIs e das funções principais implementadas nas stores/serviços do projeto.

Observação: muitas chamadas e processamentos estão centralizados em store/apiStore.js.

apiStore => fetchApiData() ​

  • Propósito: carregar os dados principais do Book (itens, cores, tamanhos, categorias) e transformar em formato consumível pela UI.
  • Importância: API crítica — falhas aqui impactam toda a aplicação.

Exemplo de response (objeto representativo):

json
{
  "tipo": "sutias",
  "cluster": "SUTOP",
  "subCategoria": "Sutop®",
  "referencia": "51522",
  "colecao": "Athena",
  "tecido": "NEO BLOCK POWER 4S + ALQUIMIA",
  "tecnologiaEBeneficios": [
    {"tecnologia":"Cós e costas em Neo Block Power 4S","beneficio":"Sensação suave"}
  ],
  "infoText": ["Com Bojo","Sem Aro"],
  "tamanhos": ["40B","40C","40D"],
  "cores": ["071 - preto","368 - toffee"]
}

apiStore => linkBuilder ​

  • Propósito: mapear referências do Book para productId da plataforma (VTEX) e montar links para prateleira infinita.
  • Fluxo: encontra productId → monta URL (ex.: https://liz.myvtex.com/assisted-sales/sales-app/product/${productId}) → redireciona no click do ItemCard.

apiStore => callLogin ​

  • Propósito: validar credenciais no backend e retornar token/usuário.

Request exemplo:

json
{
  "nome_loja": "representante",
  "password": "repliz2025"
}

Response exemplo:

json
{
  "message": "",
  "token": "",
  "dados": {"_id":"","nome_loja":"","cnpj":"","password":""}
}

Operações CRUD (uso em CrudView) ​

Página responsável por criação, leitura, atualização e remoção dos itens que compõem o Book (itensCollections no MongoDB).

Esquema de item (exemplo para create/update):

json
{
  "tipo": "",
  "cluster": "",
  "subCategoria": "",
  "referencia": "",
  "colecao": "",
  "tecido": "",
  "imagemBase64": "",
  "lstEdit": null,
  "tecnologiaEBeneficios": [],
  "infoText": [],
  "tamanhos": [],
  "cores": []
}
  • create: envia objeto acima para a API de criação.
  • read: buscar item por ID (ID concatenado na URL).
  • update: enviar objeto com _id preenchido para atualizar.
  • delete: chamar endpoint /delete/:id com o ID a ser removido.

Boas práticas:

  • Validar que tamanhos, cores e infoText sejam arrays antes de enviar.
  • Encurtar/normalizar campos de imagem quando possível (usar referências em vez de Base64 para escala).
